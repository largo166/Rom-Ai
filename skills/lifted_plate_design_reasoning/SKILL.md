# 抬板项目空间价值推导工作流

## 用途
用于住宅或复合社区中“抬板 / 地库抬升 / 下沉庭院 / 立体会所 / 展示区台地高差”类方案的设计判断、汇报论证与成果组织。

## 核心观点
抬板不是单纯抬高地库，而是重写社区地面：把车库、会所、架空层、庭院、城市界面组织为连续的立体生活系统。价值不在“高度”，而在“关系”。

## 触发场景
- 甲方质疑庭院尺度、台地高差、地库采光、下沉庭院价值
- 项目需要论证展示区庭院多大才合理
- 需要把抬板从工程动作转译为空间价值、溢价逻辑、带看记忆点
- 需要形成汇报 PPT、概念文本、空间推导图或剖面策略

## 输入
- 总图、地库边界、楼间距、庭院进深/面宽、标高关系
- 会所、架空层、下沉庭院、入口、城市界面等功能需求
- 甲方诉求：尺度、成本、带看、运营、董事长汇报逻辑
- 竞品案例、项目定位、客户画像

## 标准分析步骤
1. 定义问题：这是工程抬高、展示区造景，还是社区地面系统重构？
2. 剖面拆解：列出地库、地面、架空层、会所、下沉庭院、城市界面的垂直关系。
3. 功能串联：判断入口—庭院—会所—架空层—归家动线是否连续。
4. 四线校核：
   - 价值线：是否形成可感知溢价；
   - 成本线：是否把非销售面积转译为体验资产；
   - 报建线：是否存在标高、消防、采光、覆土风险；
   - 运营线：是否后期可持续使用。
5. 空间体验判断：光影、触觉、空间、时间四个维度是否成立。
6. 输出汇报语言：用“关系、场景、带看、运营、溢价”替代单纯“高度、面积、景观”。

## 输出模板
### 一句话判断
本项目抬板价值不在于把地库抬高，而在于通过高差重组入口、庭院、会所、架空层与城市界面的体验关系。

### 设计策略
- 超级界面：
- 垂直会所：
- 下沉庭院：
- 灰空间与架空层：
- 归家动线：
- 运营场景：

### 风险清单
- 标高与无障碍：
- 消防疏散：
- 采光通风：
- 成本与结构：
- 后期运营：

### 汇报结论
建议用“剖面系统 + 功能链路 + 价值转译 + 案例对标”的方式向甲方说明，而不是只讲庭院面积或景观效果。

## 示例提示词
“请根据这个总图和庭院尺寸，判断抬板展示区的高差是否形成空间价值。请按剖面关系、功能链路、价值转译、风险校核和对甲方汇报话术输出。”

## 边界条件
- 不把抬板简单等同于地库抬高。
- 不只用效果图美观判断价值。
- 必须同时检查成本、报建、运营和带看逻辑。
- 若缺少标高、消防或地库边界资料，应标记为待确认。
